## 概述

本指南介绍如何使用预构建的Docker镜像进行部署。这种方式适合：
- 在Linux服务器上快速部署，无需构建环境
- CI/CD自动化部署
- 多服务器统一部署

### 部署流程

```
开发者端: 构建镜像 → 推送到仓库
    ↓
Linux服务器: 拉取镜像 → 启动服务
```

---

## 快速开始

### Linux服务器一键部署

```bash
# 1. 设置镜像环境变量
export BACKEND_IMAGE="docker.io/your-username/blog-backend:latest"
export FRONTEND_IMAGE="docker.io/your-username/blog-frontend:latest"

# 2. 进入部署目录
cd deploy

# 3. 一键部署（自动检查端口、拉取镜像、启动服务）
./scripts/quick-deploy.sh
```

部署完成后，访问：
- HTTPS: `https://your-domain.com`
- HTTP: `http://your-domain.com` (自动重定向到HTTPS)

## 镜像构建和推送

### 1. 配置镜像仓库信息

```bash
cd deploy
cp .env.images.example .env.images
```

编辑 `.env.images` 文件：

```env
IMAGE_REGISTRY=docker.io
IMAGE_USERNAME=your-username
IMAGE_TAG=latest
```

### 2. 构建镜像

**Linux/macOS:**
```bash
./scripts/build-images.sh
```

**Windows:**
```cmd
scripts\build-images.cmd
```

### 3. 推送镜像到仓库

**推送前需要登录:**
```bash
# Docker Hub
docker login

# 私有仓库
docker login your-registry.com
```

**推送镜像:**
```bash
./scripts/push-images.sh
```

### 4. 验证镜像

```bash
# 查看本地镜像
docker images | grep blog

# 测试拉取（可选）
docker pull docker.io/your-username/blog-backend:latest
docker pull docker.io/your-username/blog-frontend:latest
```

## Linux服务器部署

### 方式一：一键部署（推荐）

```bash
# 设置镜像环境变量
export BACKEND_IMAGE="docker.io/your-username/blog-backend:latest"
export FRONTEND_IMAGE="docker.io/your-username/blog-frontend:latest"

# 运行一键部署脚本
cd deploy
chmod +x scripts/*.sh
./scripts/quick-deploy.sh
```

一键部署脚本会自动：
1. ✅ 检查80/443端口占用
2. ✅ 拉取镜像
3. ✅ 启动服务

### 方式二：分步部署

#### 步骤1: 检查端口

```bash
./scripts/check-ports.sh
```

如果端口被占用，脚本会提示处理方案。

#### 步骤2: 拉取镜像并启动

```bash
export BACKEND_IMAGE="docker.io/your-username/blog-backend:latest"
export FRONTEND_IMAGE="docker.io/your-username/blog-frontend:latest"
./scripts/pull-and-run.sh
```

### 方式三：手动部署

```bash
# 1. 设置环境变量
export BACKEND_IMAGE="docker.io/your-username/blog-backend:latest"
export FRONTEND_IMAGE="docker.io/your-username/blog-frontend:latest"
export APP_PORT=80
export APP_SSL_PORT=443
export ADMIN_PASSWORD=your-password
export DOMAIN=your-domain.com

# 2. 拉取镜像
docker pull $BACKEND_IMAGE
docker pull $FRONTEND_IMAGE

# 3. 启动服务
cd deploy
docker-compose -f docker-compose.images.yml -f docker-compose.prod.yml up -d
```

## 端口占用处理

### 自动检查

部署脚本会自动检查80/443端口占用：

```bash
./scripts/check-ports.sh
```

### 常见占用情况

#### 1. 系统nginx占用

**检测:**
```bash
systemctl status nginx
```

**处理:**
```bash
# 停止系统nginx
sudo systemctl stop nginx

# 禁用开机自启（推荐）
sudo systemctl disable nginx
```

#### 2. Docker容器占用

**检测:**
```bash
docker ps
docker port <container_name>
```

**处理:**
```bash
# 停止占用端口的容器
docker stop <container_name>

# 或删除容器
docker rm -f <container_name>
```

#### 3. Apache占用

**检测:**
```bash
systemctl status apache2  # Ubuntu/Debian
systemctl status httpd     # CentOS/RHEL
```

**处理:**
```bash
# Ubuntu/Debian
sudo systemctl stop apache2
sudo systemctl disable apache2

# CentOS/RHEL
sudo systemctl stop httpd
sudo systemctl disable httpd
```

#### 4. 其他进程占用

**查找占用进程:**
```bash
# 使用 lsof
sudo lsof -i :80
sudo lsof -i :443

# 使用 netstat
sudo netstat -tulnp | grep -E ':(80|443) '

# 使用 ss
sudo ss -tulnp | grep -E ':(80|443) '
```

**处理:**
根据进程信息，停止相应的服务或进程。

## 常见问题

### Q1: 镜像拉取失败 - "unauthorized"

**原因:** 未登录镜像仓库

**解决:**
```bash
# Docker Hub
docker login

# 私有仓库
docker login your-registry.com
```

### Q2: 端口被占用，容器无法启动

**解决:**
1. 运行端口检查脚本: `./scripts/check-ports.sh`
2. 根据提示停止占用端口的服务
3. 重新运行部署脚本

### Q3: 域名无法访问

**检查清单:**
1. ✅ DNS解析是否正确: `nslookup your-domain.com`
2. ✅ 防火墙是否开放80/443端口
3. ✅ 服务是否正常运行: `docker-compose ps`
4. ✅ SSL证书是否配置（HTTPS）

### Q4: 如何更新部署？

```bash
# 1. 拉取最新镜像
docker pull docker.io/your-username/blog-backend:latest
docker pull docker.io/your-username/blog-frontend:latest

# 2. 重启服务
cd deploy
docker-compose -f docker-compose.images.yml restart
```

### Q5: 如何查看日志？

```bash
# 查看所有服务日志
docker-compose -f docker-compose.images.yml logs -f

# 查看特定服务日志
docker-compose -f docker-compose.images.yml logs -f backend
docker-compose -f docker-compose.images.yml logs -f nginx
```

### Q6: 如何备份数据？

```bash
# 备份数据库
cp backend/accounts.db backup/accounts_$(date +%Y%m%d).db
```

### Q7: 使用私有镜像仓库

**配置:**
```env
# .env.images
IMAGE_REGISTRY=your-registry.com
IMAGE_USERNAME=your-org
IMAGE_TAG=latest
```

**登录:**
```bash
docker login your-registry.com
```

## 镜像仓库选择

### Docker Hub（默认）

- **优点:** 免费、使用简单
- **缺点:** 拉取速度可能较慢（国内）
- **配置:** `IMAGE_REGISTRY=docker.io`

### 阿里云容器镜像服务

- **优点:** 国内速度快、免费
- **配置:** `IMAGE_REGISTRY=registry.cn-hangzhou.aliyuncs.com`
- **注册:** https://cr.console.aliyun.com/

### 腾讯云容器镜像服务

- **优点:** 国内速度快
- **配置:** `IMAGE_REGISTRY=ccr.ccs.tencentyun.com`
- **注册:** https://console.cloud.tencent.com/tcr

### 私有仓库

- **优点:** 完全控制、安全性高
- **配置:** `IMAGE_REGISTRY=your-registry.com`
- **需要:** 自建或使用企业级服务

## 安全建议

1. **使用HTTPS:** 确保SSL证书正确配置
2. **强密码:** 设置强管理员密码
3. **防火墙:** 只开放必要的端口
4. **定期更新:** 及时拉取最新镜像
5. **备份数据:** 定期备份数据库文件
