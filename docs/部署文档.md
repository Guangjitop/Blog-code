## 环境准备

### 1. 系统要求
- 操作系统: Ubuntu 20.04+ / CentOS 7+ / Debian 10+
- Python: 3.8+
- 内存: 至少 512MB
- 磁盘: 至少 1GB 可用空间

### 2. 安装Python和pip

**Ubuntu/Debian:**
```bash
sudo apt update
sudo apt install -y python3 python3-pip python3-venv
```

**CentOS/RHEL:**
```bash
sudo yum install -y python3 python3-pip
```

### 3. 验证安装
```bash
python3 --version
pip3 --version
```

## 项目部署

### 1. 创建项目目录
```bash
# 创建应用目录
sudo mkdir -p /opt/account-api
cd /opt/account-api

# 设置权限
sudo chown -R $USER:$USER /opt/account-api
```

### 2. 上传项目文件
```bash
# 方法1: 使用 scp 从本地上传
scp main.py requirements.txt user@server:/opt/account-api/

# 方法2: 使用 git 克隆
git clone <your-repo-url> /opt/account-api

# 方法3: 使用 rsync
rsync -avz --progress ./ user@server:/opt/account-api/
```

### 3. 创建虚拟环境
```bash
cd /opt/account-api

# 创建虚拟环境
python3 -m venv venv

# 激活虚拟环境
source venv/bin/activate

# 升级 pip
pip install --upgrade pip
```

### 4. 安装依赖
```bash
pip install -r requirements.txt
```

### 5. 测试运行
```bash
# 测试启动
python main.py

# 在另一个终端测试访问
curl http://localhost:8000/api/stats
```

按 `Ctrl+C` 停止测试运行。

## 后台自启动配置

### 方法一: 使用 systemd (推荐)

#### 1. 创建 systemd 服务文件
```bash
sudo nano /etc/systemd/system/account-api.service
```

#### 2. 添加以下内容
```ini
[Unit]
Description=Account Management API Service
After=network.target

[Service]
Type=simple
User=www-data
Group=www-data
WorkingDirectory=/opt/account-api
Environment="PATH=/opt/account-api/venv/bin"
ExecStart=/opt/account-api/venv/bin/python main.py
Restart=always
RestartSec=10
StandardOutput=append:/var/log/account-api/app.log
StandardError=append:/var/log/account-api/error.log

# 安全配置
NoNewPrivileges=true
PrivateTmp=true

[Install]
WantedBy=multi-user.target
```

#### 3. 创建日志目录
```bash
sudo mkdir -p /var/log/account-api
sudo chown -R www-data:www-data /var/log/account-api
```

#### 4. 设置项目权限
```bash
sudo chown -R www-data:www-data /opt/account-api
```

#### 5. 启用并启动服务
```bash
# 重新加载 systemd 配置
sudo systemctl daemon-reload

# 启用开机自启动
sudo systemctl enable account-api

# 启动服务
sudo systemctl start account-api

# 查看服务状态
sudo systemctl status account-api
```

### 方法二: 使用 Supervisor

#### 1. 安装 Supervisor
```bash
# Ubuntu/Debian
sudo apt install -y supervisor

# CentOS/RHEL
sudo yum install -y supervisor
sudo systemctl enable supervisord
sudo systemctl start supervisord
```

#### 2. 创建配置文件
```bash
sudo nano /etc/supervisor/conf.d/account-api.conf
```

#### 3. 添加以下内容
```ini
[program:account-api]
command=/opt/account-api/venv/bin/python main.py
directory=/opt/account-api
user=www-data
autostart=true
autorestart=true
redirect_stderr=true
stdout_logfile=/var/log/account-api/app.log
stdout_logfile_maxbytes=50MB
stdout_logfile_backups=10
environment=PATH="/opt/account-api/venv/bin"
```

#### 4. 创建日志目录
```bash
sudo mkdir -p /var/log/account-api
sudo chown -R www-data:www-data /var/log/account-api
```

#### 5. 启动服务
```bash
# 重新加载配置
sudo supervisorctl reread
sudo supervisorctl update

# 启动服务
sudo supervisorctl start account-api

# 查看状态
sudo supervisorctl status account-api
```

### 方法三: 使用 Docker (容器化部署)

#### 1. 安装 Docker
```bash
# Ubuntu/Debian
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
sudo usermod -aG docker $USER

# 启动 Docker
sudo systemctl enable docker
sudo systemctl start docker
```

#### 2. 创建 Dockerfile
```bash
cd /opt/account-api
nano Dockerfile
```

添加内容:
```dockerfile
FROM python:3.11-slim

WORKDIR /app

# 安装依赖
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 复制应用文件
COPY main.py .

# 创建数据目录
RUN mkdir -p /app/data

EXPOSE 8000

CMD ["python", "main.py"]
```

#### 3. 创建 docker-compose.yml
```bash
nano docker-compose.yml
```

添加内容:
```yaml
version: '3.8'

services:
  account-api:
    build: .
    container_name: account-api
    restart: always
    ports:
      - "8000:8000"
    volumes:
      - ./data:/app/data
      - ./accounts.db:/app/accounts.db
    environment:
      - TZ=Asia/Shanghai
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"
```

#### 4. 启动容器
```bash
# 构建并启动
docker-compose up -d

# 查看状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

## Nginx反向代理

### 1. 安装 Nginx
```bash
# Ubuntu/Debian
sudo apt install -y nginx

# CentOS/RHEL
sudo yum install -y nginx

# 启动 Nginx
sudo systemctl enable nginx
sudo systemctl start nginx
```

### 2. 创建配置文件
```bash
sudo nano /etc/nginx/sites-available/account-api
```

### 3. 添加配置内容
```nginx
server {
    listen 80;
    server_name your-domain.com;  # 替换为你的域名或IP

    # 访问日志
    access_log /var/log/nginx/account-api-access.log;
    error_log /var/log/nginx/account-api-error.log;

    # 反向代理配置
    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # 静态文件缓存
    location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ {
        proxy_pass http://127.0.0.1:8000;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}
```

### 4. 启用配置
```bash
# Ubuntu/Debian
sudo ln -s /etc/nginx/sites-available/account-api /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重新加载 Nginx
sudo systemctl reload nginx
```

### 5. 配置 HTTPS (可选但推荐)
```bash
# 安装 Certbot
sudo apt install -y certbot python3-certbot-nginx

# 获取 SSL 证书
sudo certbot --nginx -d your-domain.com

# 自动续期测试
sudo certbot renew --dry-run
```

## 日志管理

### 1. 查看实时日志
```bash
# systemd 日志
sudo journalctl -u account-api -f

# 应用日志
tail -f /var/log/account-api/app.log

# 错误日志
tail -f /var/log/account-api/error.log

# Nginx 日志
tail -f /var/log/nginx/account-api-access.log
```

### 2. 配置日志轮转
```bash
sudo nano /etc/logrotate.d/account-api
```

添加内容:
```
/var/log/account-api/*.log {
    daily
    rotate 14
    compress
    delaycompress
    notifempty
    create 0640 www-data www-data
    sharedscripts
    postrotate
        systemctl reload account-api > /dev/null 2>&1 || true
    endscript
}
```

### 3. 手动执行日志轮转
```bash
sudo logrotate -f /etc/logrotate.d/account-api
```

## 故障排查

### 1. 服务无法启动
```bash
# 查看详细错误信息
sudo systemctl status account-api -l

# 查看完整日志
sudo journalctl -u account-api -n 100 --no-pager

# 检查端口占用
sudo lsof -i :8000
sudo netstat -tulpn | grep :8000

# 手动测试启动
cd /opt/account-api
source venv/bin/activate
python main.py
```

### 2. 权限问题
```bash
# 检查文件权限
ls -la /opt/account-api

# 修复权限
sudo chown -R www-data:www-data /opt/account-api
sudo chmod -R 755 /opt/account-api

# 检查数据库文件权限
ls -la /opt/account-api/accounts.db
sudo chown www-data:www-data /opt/account-api/accounts.db
```

### 3. 端口被占用
```bash
# 查找占用进程
sudo lsof -i :8000

# 终止进程
sudo kill -9 <PID>

# 或修改应用端口
# 编辑 main.py 最后一行
uvicorn.run(app, host="0.0.0.0", port=8001)
```

### 4. 数据库锁定
```bash
# 检查数据库文件
ls -la /opt/account-api/accounts.db

# 如果数据库损坏，恢复备份
cp /opt/account-api/accounts.db.backup /opt/account-api/accounts.db

# 重启服务
sudo systemctl restart account-api
```

### 5. 内存不足
```bash
# 查看内存使用
free -h

# 查看进程内存
ps aux | grep python

# 添加 swap (如果需要)
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

### 6. 健康检查脚本
创建监控脚本:
```bash
sudo nano /opt/account-api/health-check.sh
```

添加内容:
```bash
#!/bin/bash

SERVICE_URL="http://localhost:8000/api/stats"
LOG_FILE="/var/log/account-api/health-check.log"

if curl -f -s "$SERVICE_URL" > /dev/null 2>&1; then
    echo "[$(date)] Service is healthy" >> "$LOG_FILE"
    exit 0
else
    echo "[$(date)] Service is down, restarting..." >> "$LOG_FILE"
    systemctl restart account-api
    sleep 5
    if curl -f -s "$SERVICE_URL" > /dev/null 2>&1; then
        echo "[$(date)] Service restarted successfully" >> "$LOG_FILE"
    else
        echo "[$(date)] Service restart failed!" >> "$LOG_FILE"
    fi
    exit 1
fi
```

设置权限并添加到 crontab:
```bash
sudo chmod +x /opt/account-api/health-check.sh

# 编辑 crontab (每5分钟检查一次)
sudo crontab -e

# 添加以下行
*/5 * * * * /opt/account-api/health-check.sh
```

## 安全建议

### 1. 防火墙配置
```bash
# Ubuntu (UFW)
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable

# CentOS (firewalld)
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --reload
```

### 2. 限制直接访问
```bash
# 只允许本地访问 8000 端口
# 修改 main.py
uvicorn.run(app, host="127.0.0.1", port=8000)

# 通过 Nginx 反向代理对外提供服务
```

### 3. 定期备份
```bash
# 创建备份脚本
sudo nano /opt/account-api/backup.sh
```

添加内容:
```bash
#!/bin/bash
BACKUP_DIR="/opt/backups/account-api"
DATE=$(date +%Y%m%d_%H%M%S)

mkdir -p "$BACKUP_DIR"
cp /opt/account-api/accounts.db "$BACKUP_DIR/accounts_$DATE.db"

# 保留最近7天的备份
find "$BACKUP_DIR" -name "accounts_*.db" -mtime +7 -delete

echo "[$(date)] Backup completed: accounts_$DATE.db"
```

设置定时备份:
```bash
sudo chmod +x /opt/account-api/backup.sh

# 每天凌晨2点备份
sudo crontab -e
0 2 * * * /opt/account-api/backup.sh >> /var/log/account-api/backup.log 2>&1
```
