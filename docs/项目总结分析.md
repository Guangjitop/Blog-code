## 一、项目概述

本项目是一套基于 **FastAPI + React** 的多租户账号管理系统，主要用于账号的批量管理、分发和状态追踪。系统通过授权码实现多租户数据隔离，支持分类管理、批量导入、自动标记已使用账号等功能，并附带发货标签管理模块。

## 四、核心功能模块

### 4.1 授权码系统（多租户核心）

- 管理员生成/删除/启用/禁用授权码
- 支持自定义授权码或自动随机生成（格式: `xxxxxxxx-xxxx`）
- 授权码作为租户标识，实现数据完全隔离
- Cookie-based 认证机制

### 4.2 账号管理

- **CRUD**: 添加、删除、更新、查询账号
- **批量导入**: 支持批量添加账号
- **状态追踪**: 自动标记已使用 / 未使用
- **分类管理**: 按服务类型（Netflix、Spotify 等）分类
- **自动分发**: `/api/get-account` 自动返回未使用账号并标记
- **启用/禁用**: 账号级别的启用控制
- **数据导出**: 支持 CSV / TXT / JSON 格式导出

### 4.3 发货标签管理

- 发货标签分类管理
- 标签内容 CRUD
- 批量导入标签内容
- 使用状态追踪

### 4.4 统计与可视化

- 总体账号统计（总数、已用、未用、使用率）
- 按分类统计
- **Recharts** 饼图 + 柱状图可视化展示
- 网站访问量统计

### 4.5 首页门户

- Three.js 3D 星空动效背景
- 开发者主页展示
- TuneHub 音乐播放器集成
- 网站访问统计 API 集成

## 七、API 接口设计

### 认证方式

| 角色 | 认证方式 | 说明 |
|------|---------|------|
| 管理员 | Cookie (`admin_token`) | 密码: 环境变量 `ADMIN_PASSWORD` |
| 用户 | Cookie (`user_key`) 或 Query (`key`) | 授权码认证 |

### 主要接口分组

- **管理员接口** (`/api/admin/*`): 授权码 CRUD、全局账号管理
- **用户接口** (`/api/user/*`): 分类管理、账号管理（租户隔离）
- **公共接口** (`/api/*`): 获取账号、统计信息、密码查询
- **发货标签接口** (`/api/user/shipment/*`): 标签分类和内容管理
- **网站统计** (`/api/stats/site`): 访问量统计

## 八、部署架构

### 端口映射

```
用户请求
    │
    ▼
┌──────────────────────┐
│  Nginx (80/443)      │  ← 或宿主机 Nginx 反向代理到 8998
│  - 静态文件 (Home/)  │
│  - /app/manage/* SPA │
│  - /api/* → backend  │
└──────────┬───────────┘
           │
           ▼
┌──────────────────────┐
│  FastAPI (8998)      │  ← 容器内端口
│  - REST API          │
│  - SQLite DB         │
└──────────────────────┘
```

### 两种部署方案

| 方案 | 适用场景 | 端口 |
|------|---------|------|
| **方案 A** | 80/443 端口空闲 | Docker 直接绑定 80/443 |
| **方案 B** | 80/443 被占用 | Docker 用 8998/8999，宿主机 Nginx 反代 |

### Docker 镜像

- **Frontend**: 多阶段构建（Node 20 构建 → Nginx Alpine 运行）
- **Backend**: Python 3.11-slim 单阶段

### 自动化运维

- `Makefile` 提供 15+ 命令（deploy / up / down / logs / ssl / backup 等）
- 部署脚本自动检测端口、选择方案、申请 SSL、启动服务
- 支持 Linux (`deploy.sh`) 和 Windows (`deploy.ps1`)
- SSL 自动续期（crontab + certbot）

### 改进建议 🔧

| 维度 | 建议 |
|------|------|
| **密码安全** | 当前明文存储密码，建议使用 bcrypt/argon2 哈希 |
| **认证增强** | 管理员密码硬编码，建议使用 JWT + 安全的会话管理 |
| **后端拆分** | `main.py` 已达 2100+ 行，建议按模块拆分（routes / models / services） |
| **数据库升级** | SQLite 适合小规模，高并发场景可考虑 PostgreSQL |
| **前端测试覆盖** | 当前仅 1 个属性测试文件，建议增加页面和组件的单元/集成测试 |
| **环境变量管理** | 管理员密码等敏感信息应统一通过环境变量 / Secret 管理 |
| **日志系统** | 后端缺少结构化日志，建议引入 logging + JSON 格式 |
| **速率限制** | API 接口缺少限流保护，建议添加 rate limiting 中间件 |
| **输入校验** | 部分接口使用 GET + Query 参数修改数据，建议改为 POST/PUT + Body |
| **错误处理** | 前端错误处理可更统一，建议全局 Error Boundary |

## 十一、总结

本项目是一个**功能完整、部署方案成熟**的中小型 Web 应用。主要亮点：

1. **完整的多租户体系** — 授权码隔离数据，管理员与用户权限清晰分离
2. **现代化前端** — React 19 + TypeScript + Tailwind + Radix UI，UI 精致（Glassmorphism 风格）
3. **部署自动化程度高** — Docker + Makefile + 一键脚本，覆盖 Linux/Windows 双平台
4. **文档丰富** — 部署、修复、升级文档齐全，降低运维门槛
5. **轻量级架构** — SQLite + 单文件后端，适合中小规模使用场景

主要风险点在于**密码明文存储**和**后端单文件架构的可维护性**，建议在后续版本中优先解决这两个问题。整体而言，项目完成度高，已具备生产环境部署和使用能力。
