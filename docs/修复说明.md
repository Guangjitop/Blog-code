# Linux 部署脚本修复说明

## 问题诊断

根据您提供的错误截图，部署脚本在 Linux 服务器上启动失败。主要问题包括:

1. ❌ **Python 命令硬编码**: 脚本使用 `python3`，但某些系统可能只有 `python`
2. ❌ **缺少依赖检查**: 启动前未验证必要的 Python 模块是否已安装
3. ❌ **交互式提示**: `logs` 命令包含交互式输入，不适合自动化部署
4. ❌ **错误处理不足**: `set -e` 导致脚本在非关键错误时退出
5. ❌ **缺少诊断信息**: 启动失败时未显示详细错误信息

## 已修复的问题

### 1. 自动检测 Python 命令 ✅

**修复前**:
```bash
nohup python3 "$MAIN_FILE" > "$LOG_FILE" 2> "$ERROR_LOG_FILE" &
```

**修复后**:
```bash
# 自动检测可用的 Python 命令
detect_python() {
    if command -v python3 &> /dev/null; then
        echo "python3"
    elif command -v python &> /dev/null; then
        if python -c 'import sys; exit(0 if sys.version_info >= (3, 7) else 1)' 2>/dev/null; then
            echo "python"
        else
            print_error "需要 Python 3.7 或更高版本"
            exit 1
        fi
    else
        print_error "未找到 Python，请先安装 Python 3.7+"
        exit 1
    fi
}

PYTHON_CMD=$(detect_python)
nohup $PYTHON_CMD "$MAIN_FILE" > "$LOG_FILE" 2> "$ERROR_LOG_FILE" &
```

**优势**:
- ✅ 兼容不同的 Linux 发行版
- ✅ 自动选择正确的 Python 命令
- ✅ 验证 Python 版本是否满足要求

### 2. 添加依赖检查 ✅

**新增功能**:
```bash
check_dependencies() {
    print_info "检查依赖..."
    
    # 检查main.py是否存在
    if [ ! -f "$MAIN_FILE" ]; then
        print_error "未找到 main.py 文件: $MAIN_FILE"
        exit 1
    fi
    
    # 检查Python模块
    if ! $PYTHON_CMD -c "import fastapi, uvicorn" 2>/dev/null; then
        print_warning "缺少必要的Python模块"
        print_info "正在安装依赖..."
        
        if [ -f "$PROJECT_DIR/requirements.txt" ]; then
            $PYTHON_CMD -m pip install -r "$PROJECT_DIR/requirements.txt" || {
                print_error "依赖安装失败，请手动运行: $PYTHON_CMD -m pip install -r requirements.txt"
                exit 1
            }
        else
            print_error "未找到 requirements.txt 文件"
            exit 1
        fi
    fi
    
    print_info "依赖检查完成"
}
```

**优势**:
- ✅ 启动前自动检查依赖
- ✅ 缺少依赖时自动安装
- ✅ 提供清晰的错误提示

### 3. 移除交互式提示 ✅

**修复前**:
```bash
print_info "是否实时查看日志? (y/n)"
read -r answer
if [ "$answer" = "y" ] || [ "$answer" = "Y" ]; then
    tail -f "$LOG_FILE"
fi
```

**修复后**:
```bash
# 提供实时查看的命令提示
echo ""
print_info "实时查看日志命令:"
echo "  tail -f $LOG_FILE"
if [ -f "$ERROR_LOG_FILE" ]; then
    echo "  tail -f $ERROR_LOG_FILE"
fi
```

**优势**:
- ✅ 适合自动化部署和 CI/CD
- ✅ 不会阻塞脚本执行
- ✅ 仍然提供有用的命令提示

if ps -p "$PID" > /dev/null 2>&1; then

## 使用方法

### 方法一: 快速部署 (推荐)

```bash
# 1. 上传项目到服务器
cd /path/to/message

# 2. 赋予执行权限
chmod +x scripts/quick-deploy.sh

# 3. 运行一键部署
bash scripts/quick-deploy.sh
```

### 方法二: 手动部署

```bash
# 1. 安装依赖
python3 -m pip install -r requirements.txt

# 2. 赋予脚本执行权限
chmod +x scripts/deploy.sh

# 3. 启动服务
./scripts/deploy.sh start
```
