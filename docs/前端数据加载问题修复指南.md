# 前端数据加载问题修复指南

## 问题诊断

前端无法正确加载数据的主要原因是 **API 连接配置问题**。

### 快速诊断步骤

1. **打开诊断页面**
   ```
   访问: http://你的IP/diagnose.html
   ```
   这个页面会自动测试各种 API 连接方式，并给出推荐配置。

2. **检查浏览器控制台**
   - 按 F12 打开开发者工具
   - 查看 Console 标签页的错误信息
   - 查看 Network 标签页的请求状态

3. **检查后端服务**
   ```bash
   # 检查后端是否运行
   ps aux | grep main.py
   
   # 检查端口是否监听
   lsof -i:8000
   
   # 测试后端 API
   curl http://localhost:8000/api/stats
   ```

## 解决方案

### 方案 1: 配置 Nginx 反向代理（推荐）

这是最佳方案，可以避免跨域问题，并且前端代码无需修改。

#### 步骤 1: 修改 Nginx 配置

在宝塔面板中：
1. 进入 **网站** -> 选择你的网站 -> **设置**
2. 点击 **配置文件**
3. 在 `server` 块中添加以下配置：

```nginx
# API 反向代理
location /api/ {
    proxy_pass http://127.0.0.1:8000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
}

location /admin/ {
    proxy_pass http://127.0.0.1:8000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
}

location /docs {
    proxy_pass http://127.0.0.1:8000;
    proxy_set_header Host $host;
}

location /openapi.json {
    proxy_pass http://127.0.0.1:8000;
}
```

#### 步骤 2: 重启 Nginx

```bash
# 测试配置
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx
```

#### 步骤 3: 验证

访问前端页面，数据应该能正常加载了。

---

### 方案 2: 直接访问后端（临时方案）

如果不想配置 Nginx，可以直接通过 8000 端口访问：

```
访问: http://你的IP:8000/index.html
```

**注意**: 需要确保：
1. 防火墙开放 8000 端口
2. 将 `index.html` 复制到项目根目录

```bash
# 开放端口（如果使用防火墙）
sudo firewall-cmd --permanent --add-port=8000/tcp
sudo firewall-cmd --reload
```

---

### 方案 3: 修改前端配置（不推荐）

如果以上方案都不可行，可以修改前端代码直接指向后端地址。

在 `index.html` 中找到这一行：
```javascript
const API_BASE = ref(getApiBase());
```

替换为：
```javascript
const API_BASE = ref('http://你的IP:8000');
```

**缺点**: 
- 存在跨域问题
- 需要确保后端 CORS 配置正确
- 每次更换服务器需要修改代码

## 常见错误及解决方法

### 错误 1: Network Error / ERR_CONNECTION_REFUSED

**原因**: 后端服务未运行或端口不通

**解决**:
```bash
# 启动后端
cd /www/wwwroot/你的项目路径
python main.py

# 或使用后台运行
nohup python main.py > backend.log 2>&1 &
```

### 错误 2: CORS Error

**原因**: 跨域请求被阻止

**解决**: 
1. 使用 Nginx 反向代理（方案 1）
2. 或确保后端 `main.py` 中 CORS 配置正确：
```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

### 错误 3: 404 Not Found

**原因**: API 路径不正确

**解决**:
1. 检查 Nginx 配置中的 `proxy_pass` 路径
2. 确保后端路由正确

### 错误 4: 数据格式错误

**原因**: 后端返回的数据格式不符合前端预期

**解决**:
1. 打开浏览器控制台查看实际返回的数据
2. 检查后端 API 是否正常返回数组格式

修复后，按以下步骤验证：

1. **清除浏览器缓存**
   - Chrome: Ctrl + Shift + Delete
   - 或使用无痕模式: Ctrl + Shift + N

2. **刷新页面**

3. **检查数据**
   - 统计卡片应显示正确数字
   - 账号列表应显示数据
   - 分类列表应显示数据

4. **测试功能**
   - 添加账号
   - 添加分类
   - 删除操作
   - 重置操作

## 推荐配置总结

### 生产环境（推荐）

```
前端: Nginx (80端口) -> 反向代理 -> 后端 (8000端口)
访问: http://你的IP
```

**优点**:
- 无跨域问题
- 统一入口
- 可以配置 HTTPS
- 性能更好

### 开发/测试环境

```
直接访问后端: http://你的IP:8000
```

**优点**:
- 配置简单
- 便于调试
