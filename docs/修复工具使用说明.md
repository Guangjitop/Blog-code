# 前端数据加载问题 - 修复工具使用说明

## 问题描述

前端页面显示"暂无账号数据"，但后端数据库中有数据。这是由于前端无法正确连接到后端 API 导致的。

## 快速修复（3 步）

### 第 1 步：运行诊断脚本

根据你的操作系统选择：

**Linux / macOS:**
```bash
bash quick_fix.sh
```

**Windows:**
```powershell
powershell -ExecutionPolicy Bypass -File quick_fix.ps1
```

脚本会自动检查：
- ✓ 后端服务是否运行
- ✓ API 是否响应
- ✓ 文件权限是否正确
- ✓ Nginx 配置是否存在

### 第 2 步：查看诊断页面

1. 确保后端服务正在运行
2. 在浏览器中打开：`http://你的IP:8000/diagnose.html`
3. 页面会自动测试各种连接方式
4. 根据测试结果选择最佳配置

### 第 3 步：应用修复方案

根据诊断结果，选择以下方案之一：

#### 方案 A：配置 Nginx 反向代理（推荐）

1. 参考 `nginx_config_example.conf`
2. 在宝塔面板或 Nginx 配置文件中添加反向代理规则
3. 重启 Nginx：`systemctl restart nginx`
4. 访问前端：`http://你的IP`

#### 方案 B：直接访问后端（临时）

1. 确保防火墙开放 8000 端口
2. 直接访问：`http://你的IP:8000/index.html`

## 常见问题

### Q1: 运行脚本后仍然无法加载数据？

**A:** 
1. 查看 `diagnostic_report.txt` 了解详细信息
2. 打开浏览器控制台（F12）查看错误信息
3. 确认后端服务正在运行：`ps aux | grep main.py`

### Q2: 如何确认后端服务正在运行？

**A:**
```bash
# 检查进程
ps aux | grep main.py

# 检查端口
lsof -i:8000

# 测试 API
curl http://localhost:8000/api/stats
```

### Q3: Nginx 反向代理配置在哪里？

**A:**
- 宝塔面板：网站 -> 设置 -> 配置文件
- 手动配置：`/etc/nginx/sites-enabled/` 或 `/etc/nginx/conf.d/`

### Q4: 如何查看后端日志？

**A:**
```bash
# 如果使用 nohup 启动
tail -f backend.log

# 如果使用 systemd
journalctl -u account-manager -f
```

### Q5: 浏览器显示 CORS 错误？

**A:**
1. 使用 Nginx 反向代理（推荐）
2. 或确保后端 `main.py` 中 CORS 配置正确

## 推荐配置

### 生产环境

```
架构: 前端(Nginx:80) -> 反向代理 -> 后端(FastAPI:8000)
访问: http://你的域名
优点: 统一入口、无跨域、可配置 HTTPS
```

### 开发环境

```
架构: 直接访问后端
访问: http://localhost:8000
优点: 配置简单、便于调试
```
