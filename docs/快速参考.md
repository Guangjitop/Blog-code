# 🚀 Linux 部署快速参考

## 一键部署 (推荐)

```bash
cd /path/to/message
chmod +x scripts/quick-deploy.sh
bash scripts/quick-deploy.sh
```

## 手动部署

```bash
# 1. 安装依赖
python3 -m pip install -r requirements.txt

# 2. 启动服务
chmod +x scripts/deploy.sh
./scripts/deploy.sh start
```

## 服务管理命令

| 命令 | 说明 |
|------|------|
| `./scripts/deploy.sh start` | 启动服务 |
| `./scripts/deploy.sh stop` | 停止服务 |
| `./scripts/deploy.sh restart` | 重启服务 |
| `./scripts/deploy.sh status` | 查看状态 |
| `./scripts/deploy.sh logs` | 查看日志 |
| `./scripts/deploy.sh errors` | 查看错误日志 |

## 常见问题速查

### ❌ 未找到 Python

**Ubuntu/Debian**:
```bash
sudo apt update && sudo apt install python3 python3-pip -y
```

**CentOS/RHEL**:
```bash
sudo yum install python3 python3-pip -y
```

### ❌ 依赖安装失败

```bash
python3 -m pip install -r requirements.txt
# 或
python3 -m pip install --user -r requirements.txt
```

### ❌ externally-managed-environment 错误

**问题**: Ubuntu 22.04+、Debian 12+ 等系统的 PEP 668 保护

**解决方案 1 (推荐 - 自动处理)**:
```bash
# 最新脚本已自动处理，直接运行即可
bash scripts/quick-deploy.sh
```

**解决方案 2 (手动)**:
```bash
# 使用 --break-system-packages
python3 -m pip install --break-system-packages -r requirements.txt
```

**解决方案 3 (生产环境推荐)**:
```bash
# 使用虚拟环境
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python main.py
```

### ❌ 权限不足

```bash
chmod +x scripts/deploy.sh
chmod +x scripts/quick-deploy.sh
```

### ❌ 端口被占用

```bash
# 查看占用进程
lsof -i :8000
# 终止进程
kill -9 <PID>
```

### ❌ 启动失败

```bash
# 查看详细错误
./scripts/deploy.sh errors
# 或
cat error.log
```

## 验证部署

```bash
# 访问文档
# 浏览器打开: http://your-server-ip:8000/docs
```

## 日志查看

```bash
# 查看最近日志
./scripts/deploy.sh logs

# 实时监控
tail -f app.log

# 查看错误
tail -f error.log
```

## 生产环境 (systemd)

```bash
# 创建服务文件
sudo nano /etc/systemd/system/account-api.service

# 重载配置
sudo systemctl daemon-reload

# 启动服务
sudo systemctl start account-api

# 开机自启
sudo systemctl enable account-api

# 查看状态
sudo systemctl status account-api
```

## 防火墙配置

**UFW (Ubuntu)**:
```bash
sudo ufw allow 8000/tcp
sudo ufw enable
```

**firewalld (CentOS)**:
```bash
sudo firewall-cmd --permanent --add-port=8000/tcp
sudo firewall-cmd --reload
```

## 备份数据库

```bash
# 手动备份
cp accounts.db accounts-backup-$(date +%Y%m%d).db

# 查看备份
ls -lh accounts-backup-*.db
```
