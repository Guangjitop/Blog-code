## 可能遇到的问题

### ❌ 问题 1: 未找到 Python

**症状**: `未找到 Python，请先安装 Python 3.7+`

**解决**:
```bash
# Ubuntu/Debian
sudo apt update && sudo apt install python3 python3-pip -y

# CentOS/RHEL
sudo yum install python3 python3-pip -y
```

- [ ] 已安装 Python
- [ ] Python 版本 ≥ 3.7

### ❌ 问题 2: externally-managed-environment

**症状**: `error: externally-managed-environment`

**解决**: 脚本已自动处理，如果仍然失败：
```bash
python3 -m pip install --break-system-packages -r requirements.txt
./scripts/deploy.sh start
```

- [ ] 依赖安装成功
- [ ] 服务启动成功

### ❌ 问题 3: 权限不足

**症状**: `Permission denied`

**解决**:
```bash
chmod +x scripts/deploy.sh
chmod +x scripts/quick-deploy.sh
```

- [ ] 脚本有执行权限

### ❌ 问题 4: 端口被占用

**症状**: `Address already in use`

**解决**:
```bash
# 查看占用进程
lsof -i :8000

# 终止进程
kill -9 <PID>
```

- [ ] 端口 8000 可用
- [ ] 服务成功监听端口

## 生产环境额外步骤

### 配置 systemd 服务

```bash
# 创建服务文件
sudo nano /etc/systemd/system/account-api.service

# 启用服务
sudo systemctl daemon-reload
sudo systemctl enable account-api
sudo systemctl start account-api
```

- [ ] systemd 服务已配置
- [ ] 服务已启用开机自启
- [ ] 服务运行正常

### 配置 Nginx 反向代理

```bash
# 创建 Nginx 配置
sudo nano /etc/nginx/sites-available/account-api

# 启用配置
sudo ln -s /etc/nginx/sites-available/account-api /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

- [ ] Nginx 已安装
- [ ] 反向代理已配置
- [ ] 可以通过域名访问

### 配置 HTTPS

```bash
# 安装 certbot
sudo apt install certbot python3-certbot-nginx -y

# 获取证书
sudo certbot --nginx -d your-domain.com
```

- [ ] SSL 证书已安装
- [ ] HTTPS 可以访问
- [ ] 自动续期已配置

### 配置防火墙

```bash
# UFW (Ubuntu/Debian)
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable

# firewalld (CentOS/RHEL)
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --reload
```

- [ ] 防火墙已配置
- [ ] HTTP/HTTPS 端口已开放

### 配置日志轮转

```bash
# 创建 logrotate 配置
sudo nano /etc/logrotate.d/account-api
```

- [ ] 日志轮转已配置
- [ ] 测试轮转正常

### 配置监控

```bash
# 创建健康检查脚本
sudo nano /usr/local/bin/check-account-api.sh
sudo chmod +x /usr/local/bin/check-account-api.sh

# 添加到 crontab
sudo crontab -e
# 添加: */5 * * * * /usr/local/bin/check-account-api.sh
```

- [ ] 健康检查脚本已创建
- [ ] cron 任务已配置
- [ ] 监控正常运行

### 配置备份

```bash
# 创建备份脚本
sudo nano /usr/local/bin/backup-account-db.sh
sudo chmod +x /usr/local/bin/backup-account-db.sh
```
